# apple-stock-data-pipeline
Cloud-Based Apple Stock Data Engineering Pipeline using Python, Snowflake &amp; Power BI

Excited to share my latest project: a Cloud-Based Apple Stock Data Engineering Pipeline built with Python, Snowflake, and Power BI!
I started this project to get hands-on experience with real-world data engineering — from raw API data all the way to a polished BI dashboard.
What I built:

An ETL pipeline using Python & Pandas that pulls Apple (AAPL) stock data from the Alpha Vantage API and Yahoo Finance, combining 45+ years of historical data (11,470+ records, 1980–2026)

A Snowflake cloud data warehouse to store and structure the data

An interactive Power BI dashboard with custom DAX measures — moving averages, year-over-year returns, 52-week high/low, and KPI tracking
 
Secure credential management using environment variables, following production-grade security practices
Key learnings:

 Designing modular ETL scripts (extract, transform, load) for maintainability

 Handling messy real-world data from multiple sources with different formats

 Writing DAX measures for time-series financial analysis

 Why hardcoding API keys/passwords is a major security risk — and how to avoid it
This project pushed me to think like a data engineer — not just writing code that works, but building something secure, scalable, and analysis-ready.
